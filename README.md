# 新入社員研修 — 開発プロセスとソフトウェア開発技術

組込みソフトウェア開発部門に配属される新入社員向けの講義資料です。

## 概要

| 項目 | 内容 |
|------|------|
| 対象 | 機械系学部卒の新入社員（プログラミング経験は限定的） |
| 時間 | 60分 |
| 形式 | スライド講義 + 各セクションで問いかけ |

## ファイル構成

```
.
├── slides.md         # Marpスライド本体
├── speaker_notes.md  # スピーカーノート（話す内容・問いかけ・流れ）
└── README.md
```

## 内容

| セクション | テーマ | 時間 |
|-----------|--------|------|
| 01 | ソフトウェアはなぜ壊れるのか | 5分 |
| 02 | 開発プロセスの全体像（ウォーターフォール・アジャイル・V字モデル・V&V） | 15分 |
| 03 | 開発技術の基礎（Git・テスト・コードレビュー） | 20分 |
| 04 | 品質とプロセスの関係（A-SPICE・技術的負債） | 10分 |
| 05 | 現場で活躍するためのマインドセット | 5分 |
| Q&A | — | 5分 |

## スライドの表示方法

[Marp](https://marp.app/) を使用しています。

```bash
# CLIでHTMLに変換
npm install -g @marp-team/marp-cli
marp slides.md --html --no-stdin -o slides.html
```

または [Marp for VS Code](https://marketplace.visualstudio.com/items?itemName=marp-team.marp-vscode) 拡張機能でプレビューできます。

## 設計方針

各スライドは「痛み → なぜ → 解決策」の流れで口頭で話すことを前提に設計しています。スライド自体にはキーワード・図・箇条書きだけを載せ、説明はスピーカーノートに記載しています。

機械系出身者が「あ、それと同じだ」と思えるたとえを随所に入れています（設変管理台帳・図面と検査基準・SOPなど）。
